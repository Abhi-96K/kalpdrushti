Problem Statement

Creating good videos manually is slow, costly, and skill-heavy.

For creators, students, educators, and marketers, turning an idea into a video usually means writing a script, collecting visuals, recording voice, adding subtitles, syncing music, and editing everything together.

1

Time consuming

Video production requires multiple separate tools and repeated manual effort.

2

Requires technical skills

Editing, voiceover handling, and rendering are not easy for beginners.

3

Not scalable

Making many short videos for social media or learning content becomes inefficient.

01

A system that can automatically convert one prompt into a complete video pipeline with minimal human effort.

Solution Overview

Kalpdrushti AI automates the entire prompt-to-video journey.

The user writes a prompt, selects preferences, and the backend generates a structured script, scene visuals, narration, subtitles, transitions, mood-based background music, and the final MP4 video.

Workflow

How the generation pipeline works

1
Frontend sends POST /generate
User data is sent to the backend as multipart/form-data.
2
Database job row is created
The system records a new generation job with initial pending status.
3
Structured script is generated
The orchestrator creates scene-wise script content.
4
Images and narration are generated
Visuals and voice are prepared for each scene.
5
Subtitles and final rendering
The final MP4 is built with effects, subtitles, voice, and music.
06

Job completion

After rendering, the video is marked as completed, receives a media URL, and becomes available on the dashboard for the user to view or download.

API Design

Main endpoints used by the application

Endpoint
Purpose
POST /generate
Starts a new video generation job
GET /videos
Returns all generated jobs for dashboard history
GET /videos/{video_id}
Checks current status and final video URL
POST /abort/{video_id}
Requests cancellation of a running job
07

Why API separation matters

This design keeps the frontend simple and lets the backend handle job control, progress tracking, and final output delivery in a clean way.

Database & Status Tracking

Each generation request is handled like a job

The database stores prompt data, status, video URL, and timestamps so the system can track each request from start to finish.

DB

Stored data

Processing visibility

Status values

pending

Job is created but processing has not started yet.

generating_script → generating_images → generating_audio → rendering_video

Pipeline moves through each core content generation stage.

completed / failed / abort_requested / aborted

Represents the final outcome of the job.

Core Pipeline Modules

Separated into specialized backend components

S
script_generator.py
Transforms user prompt into structured scene-wise video script.
I
image_generator.py
Creates or enhances scene images through the selected image provider.
V
voice_generator.py
Generates narration audio and subtitle files using TTS flow.
R
video_builder.py
Composes visuals, audio, music, subtitles, and motion into final MP4.
O
orchestrator.py
Coordinates the full pipeline and updates job status step by step.

Security & Production Readiness

Important checks before public deployment

!

Do not expose secrets

API keys and sensitive configuration must stay in backend environment variables only.

✓

Verify OAuth on backend

Frontend login state alone is not enough for secure production use.

🛡

Restrict CORS

Only the real frontend domain should be allowed in public deployment.

DB

Scale storage

Local SQLite and filesystem are useful for prototype stage but should be upgraded for serious multi-user production.
